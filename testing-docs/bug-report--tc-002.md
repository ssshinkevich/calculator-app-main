# Bug-report: TC-002

**Заголовок:** Неверный расчет приоритета операций

**Описание:** При вводе арифметического выражения с разными операторами ( сложение и умножение ) нарушается порядок действий. При нажатии на оператора "х" происходит принудительное вычисление предыдущей суммы. Последующий вывод работает некорректно.

**Предусловие:** 
1. Приложение открыто
2. На дисплее отображается 0

**Шаги воспроизведения:**
1. Нажать кнопку "5"
2. Нажать кнопку "+" ( сложение )
3. Нажать кнопку "5"
4. Нажать кнопку "х" ( умножение )
5. Нажать кнопку "5"
6. Нажать кнопку "="

**Фактический результат:** На дисплее отображается число 5 ( при нажатии на "x" происходит промежуточное вычисление суммы "10" )

**Ожидаемый результат:** На дисплее отображается число 30 ( сначала выполняется "5 х 5", затем "+ 5" )

**Окружение:**
- OS: MacOS Tahoe
- Browser: Google Chrome ( latest )
- Resolution: 1600 x 900

**Серьезность:** Critical ( Фундаментальная ошибка логики вычислений )

**Приоритет:** Critical ( Требует немедленного исправления )
